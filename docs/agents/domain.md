# Domain Docs

This is a single-context project.

Domain language lives in:

```text
CONTEXT.md
```

Architectural decisions live in:

```text
docs/adr/
```

Skills should read `CONTEXT.md` before proposing names, interfaces, tests, issues, PRDs, or architectural changes. ADRs should be respected unless there is strong evidence that a decision should be revisited.

`CONTEXT.md` is a glossary, not a technical specification. It should define domain terms without implementation details.
