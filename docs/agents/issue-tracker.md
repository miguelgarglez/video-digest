# Issue Tracker

This project currently uses local markdown issues under `.scratch/`.

Use this convention until a GitHub remote exists:

```text
.scratch/
  <feature-or-initiative>/
    001-short-title.md
    002-short-title.md
```

Each issue should be a vertical slice that is independently understandable and verifiable.

When publishing issues:

- prefer many small tracer-bullet slices over broad layer-based tasks;
- keep acceptance criteria concrete;
- mark blocked work explicitly;
- do not assume a GitHub issue number exists.

If the project later moves to GitHub Issues, update this file and the `## Agent skills` block in `AGENTS.md`.
