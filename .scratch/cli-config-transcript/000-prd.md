# PRD: CLI Configuration + Transcript-Only Mode

Status: done
Category: enhancement

## Problem Statement

Miguel can now install and run `video-digest` globally, but a global command no longer
implicitly sees the repository `.env` when launched from another directory. If
`OPENCODE_API_KEY` is missing, the current **Ingestion** fails after successfully
fetching and scoring the **Transcript**, which wastes useful work and makes the CLI
feel like an all-or-nothing tool.

For a future public package, the CLI also needs a clearer configuration story:
summarization should require an OpenCode token, but transcript retrieval should remain
useful without one.

## Solution

Add a transcript-only command:

```sh
video-digest transcript <youtube-url>
video-digest transcript <youtube-url> --json
```

This command fetches a **Transcript**, scores **Transcript Quality**, and writes
transcript + metadata artifacts without calling OpenCode and without requiring
`OPENCODE_API_KEY`.

Keep `video-digest ingest <youtube-url>` as the full **Digest** command. When it fails
because `OPENCODE_API_KEY` is missing, the CLI should explain that digest generation
requires the token and suggest `video-digest transcript <url>` as the token-free
fallback.

Update `doctor` so it distinguishes transcript readiness from digest readiness.

## User Stories

1. As Miguel, I want `video-digest transcript <url>`, so that I can fetch and save a
   **Transcript** without an OpenCode token.
2. As Miguel, I want transcript-only mode to score **Transcript Quality**, so that I can
   know whether the transcript is useful.
3. As Miguel, I want transcript-only mode to write metadata, so that future tools and
   agents can consume the result programmatically.
4. As Miguel, I want `video-digest transcript <url> --json`, so that Codex can fetch a
   transcript and read artifact paths without scraping human output.
5. As Miguel, I want `video-digest ingest <url>` to explain missing
   `OPENCODE_API_KEY`, so that I know whether to configure a token or use transcript
   mode.
6. As Miguel, I want `doctor` to show transcript readiness separately from digest
   readiness, so that missing OpenCode configuration does not imply the whole CLI is
   unusable.
7. As a future package user, I want transcript-only mode to work before I configure
   summarization, so that the CLI is useful immediately.

## Implementation Decisions

- Add `transcript` to the CLI command model.
- Add a focused transcript-only orchestration function rather than branching inside
  full **Ingestion**.
- Reuse the existing `TranscriptSource` and `scoreTranscriptQuality` modules.
- Add a writer for transcript-only artifacts that produces transcript JSON and
  metadata JSON, but no **Digest** Markdown and no email preview.
- Keep `OPENCODE_API_KEY` required for full `ingest`.
- Improve `missing-api-key` errors with a command suggestion.
- Update `doctor` checks so the OpenCode token check is a digest capability check.
- Do not implement `video-digest config set` in this slice.
- Do not auto-load the repository `.env` from the linked package in this slice.

## Testing Decisions

Tests should verify public behavior through CLI commands and output files.

Test targets:

- Parser support for `transcript <url>` and `transcript <url> --json`.
- Transcript-only orchestration with fake `TranscriptSource`.
- Output writer behavior for transcript-only artifacts.
- Human and JSON CLI output for transcript-only mode.
- Missing OpenCode token error messaging for full `ingest`.
- Doctor output showing transcript and digest readiness separately.

Automated tests should not call YouTube or OpenCode. A final manual smoke run may use
a real YouTube URL locally to verify transcript-only behavior.

## Out of Scope

- `video-digest config set`.
- Secure local credential storage.
- Public npm publishing.
- Gmail delivery.
- Playlist polling.
- Audio transcription fallback.
- Auto-loading the linked repository `.env`.

## Further Notes

For Miguel's local setup, exporting `OPENCODE_API_KEY` in the shell remains the
simplest way to make global `video-digest ingest` work. The transcript-only command
keeps the CLI useful even when that token is absent.
